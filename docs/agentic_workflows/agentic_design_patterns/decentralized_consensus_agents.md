# Decentralized Consensus Agents (agentic design pattern)

**Decentralized Consensus Agents** is an agentic *design pattern*: a reusable way to **reach agreement without a single
permanent hub**—local agents exchange beliefs and evidence until a forecast or allocation **converges** under explicit
stopping rules—before you pick a concrete product story (auction, hiring, logistics, etc.).

In this layout, **regional peers** publish and revise; transport and semantics carry merge semantics so oscillation and
gaming are handled deliberately, not by hope.

Coffee Agntcy’s **regional demand** story is the table vignette: many partial views still have to add up to **one number
the firm can execute**.
