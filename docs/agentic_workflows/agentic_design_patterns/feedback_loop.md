# Feedback Loop (agentic design pattern)

**Feedback Loop** is an agentic *design pattern*: a reusable way to fold **human or system feedback**—scores, disputes,
delivery outcomes—back into **routing and selection policy** so the next cycle behaves less like amnesia—before you pick
a concrete product story (auction, hiring, logistics, etc.).

In this layout, feedback lands with **attribution** (which supplier, which lane, which agent version), aggregates into
features or parameters, and flows through workflow gates that prevent gaming and cold-start silliness.

**Buyer satisfaction** in Coffee Agntcy should nudge **who gets the next call**, not decorate a slide nobody reads.
